# Weight Tracking App

A self-hosted web app for logging gym workouts. Track exercises, sets, reps, and weight over time, with progress charts per exercise.

## Features

- Log workouts with exercises, sets, reps, and weight
- Progress charts showing max weight or volume over time per exercise
- Shared exercise library across all users
- Multi-user support with admin and regular user roles
- Admins can create, edit, and reset passwords for other users

## Requirements

- Node.js 18 or higher
- Linux (for the systemd service install) or any OS for manual start

## Installation

### Production (Linux with systemd)

> **Important:** Clone into `/opt/` or another system directory — not your home directory. The systemd service runs as a dedicated system user that cannot access home directories.

```bash
sudo git clone https://github.com/zercsy/WeightTrackingApp.git /opt/WeightTrackingApp
cd /opt/WeightTrackingApp
sudo ./scripts/setup.sh
```

This will:
- Install server and client dependencies
- Generate a `server/.env` with a random JWT secret
- Run database migrations and create the default admin account
- Build the Vue frontend
- Install and start a `weight-tracker` systemd service on port 3000

The app will be available at `http://your-server-ip:3000`.

To update after pulling new changes:

```bash
cd /opt/WeightTrackingApp
sudo git pull
sudo ./scripts/setup.sh
```

**Useful service commands:**
```bash
sudo systemctl status weight-tracker
sudo systemctl restart weight-tracker
sudo systemctl stop weight-tracker
sudo journalctl -u weight-tracker -f
```

### Non-Linux / Manual Start

```bash
git clone https://github.com/zercsy/WeightTrackingApp.git
cd WeightTrackingApp
./scripts/setup.sh        # installs deps, migrates DB, builds frontend
cd server
NODE_ENV=production node src/index.js
```

## Configuration

The setup script creates `server/.env` automatically. You can edit it to change settings:

```env
PORT=3000
NODE_ENV=production
DB_PATH=./data/weights.db
JWT_SECRET=<auto-generated>
JWT_EXPIRES_IN=7d
```

The SQLite database is stored at `server/data/weights.db`.

## First Login

After installation, log in with:

- **Username:** `admin`
- **Password:** `admin123`

You will be required to change the password immediately on first login.

## Development

```bash
# Terminal 1 — backend (hot reload)
cd server
npm install
npm run migrate
npm run dev        # runs on http://localhost:3000

# Terminal 2 — frontend (hot reload)
cd client
npm install
npm run dev        # runs on http://localhost:5173
```

The Vite dev server proxies all `/api` requests to `localhost:3000`, so both servers must be running during development.

## Usage

### Tracking Workouts

1. Go to the **Tracking** tab and click **New Workout**
2. Add a name and date, then add exercises from the shared library
3. Log sets (reps + optional weight) for each exercise
4. Workouts are saved as you go

### Exercises

The exercise library is shared across all users and managed under **Settings → Exercises**. Any user can add exercises; exercises used in existing workouts cannot be deleted.

### Reports

The **Reports** tab shows a line chart of progress for a selected exercise over time. Switch between **Max Weight** and **Volume** (total reps × weight per session) metrics, and optionally filter by date range.

### User Management (Admin only)

Under **Settings → Users**, admins can:
- Create new users (assigned a temporary password; they must change it on first login)
- Change a user's username or role
- Reset a user's password
- Delete users (cannot delete yourself or the last admin)
